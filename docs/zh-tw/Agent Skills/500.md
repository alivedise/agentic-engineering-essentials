# [AEE-500] 技能 vs. 工具

## Context

隨著智能體系統日益複雜，擴展智能體能力的兩個不同抽象層逐漸浮現：**工具**和**技能**。這兩個術語在某些框架中可互換使用，但實際上代表著有意義的不同概念。混淆兩者會導致設計不良的智能體架構，行為脆弱、難以重用且難以維護。

## Design Think

**工具**是連接智能體與外部世界的介面。它們是可執行的能力：運行程式碼、讀取檔案、呼叫 API、搜索網頁。工具是無狀態的、有副作用的，由其輸入/輸出 schema 定義。智能體決定何時以及如何呼叫它們。

**技能**是打包的行為模式，指導智能體如何處理某類問題。它們是可載入的指令集——塑造智能體推理、決策風格和特定任務類型行動序列的上下文。技能在副作用方面是無狀態的，但在智能體方法上是有狀態的。

| | 工具 | 技能 |
|--|------|------|
| 本質 | 可執行能力 | 行為指導 |
| 呼叫方 | 智能體（函數呼叫） | 執行框架或智能體（上下文注入） |
| 副作用 | 有（寫入、讀取、呼叫） | 無（僅塑造推理） |
| 可重用單元 | Schema + 實作 | 提示/指令集 |
| 範例 | `bash()`、`read_file()`、`search()` | `superpowers:debugging`、`superpowers:tdd` |

技能 SHOULD 設計為具有單一明確目的。試圖做太多事情的技能會變得和上帝物件一樣脆弱。
