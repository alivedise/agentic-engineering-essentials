# [AEE-400] 工具使用與執行

## 情境

400 系列是模型從「產生文字」轉向「採取行動」的分水嶺。這個系列涵蓋代理程式呼叫外部能力的完整機制：函式呼叫 (function calling) 的協定、如何設計讓模型能正確解析的工具結構描述 (tool schema)、程式碼執行 (code execution) 的沙箱隔離 (sandboxing) 方式、瀏覽器自動化 (browser automation) 與電腦使用 (computer use) 的整合，以及工具選擇 (tool selection) 的決策邏輯和如何控制代理程式的爆炸半徑 (blast radius)。

300 系列（提示工程）的所有技術都以輸出文字為終點；400 系列的終點則是實際效果：呼叫 API、寫入資料庫、執行程式碼、提交表單、操作檔案。這個差異對安全性、成本與系統設計都至關重要。文字生成是可回溯的，工具使用 (tool use) 往往不是。對於正式上線的代理程式系統而言，理解工具如何運作、模型如何選擇工具、以及如何限制工具的影響範圍，是不可或缺的基礎知識。

## 涵蓋範圍

400 系列依「從協定到安全」的邏輯順序編排：

**協定與結構描述文章**涵蓋函式呼叫機制：
- **[AEE-401](401)** — 函式呼叫：完整的 API 協定，從工具定義到結果注入，附帶具體的 Anthropic API 範例。
- **[AEE-402](402)** — 工具結構描述設計：如何撰寫名稱、說明文字與參數結構，使模型能正確解析並呼叫工具。

**工具類型文章**涵蓋兩類高能力工具：
- **[AEE-403](403)** — 程式碼執行：沙箱直譯器、各大供應商方案，以及「模型產生的程式碼絕不能在宿主行程中直接執行」的安全要求。
- **[AEE-404](404)** — 瀏覽器自動化與電腦使用：使用 Playwright 及 browser-use 進行網頁自動化，以及 Anthropic 電腦使用 API，讓代理程式能操作完整的桌面環境。

**選擇與安全文章**涵蓋橫跨各工具的共通議題：
- **[AEE-405](405)** — 工具選擇：模型為何選錯工具、說明文字品質如何影響選擇準確率，以及管理大型工具集的策略。
- **[AEE-406](406)** — 沙箱隔離與執行安全：爆炸半徑的概念、最小權限原則在工具上的應用、隔離層設計，以及人工介入關卡（human-in-the-loop）的配置。

## 與相鄰範圍的關係

400 系列建立在 300 系列（提示工程與互動）的基礎之上。工具描述本身就是提示：指令撰寫原則（AEE-304）、結構化輸出設計（AEE-203）與提示健壯性測試（AEE-306）的技術，同樣適用於工具的說明撰寫。300 系列的技術決定模型是否「正確使用」工具；400 系列的機制則決定工具執行是否「安全」。

400 系列向上連結至 700 系列（代理程式迴圈與協作編排）。AEE-701（代理程式迴圈 / ReAct）說明工具呼叫如何嵌入多步驟推理迴圈中——觀察、推理、行動排序。AEE-401（函式呼叫）則聚焦於迴圈中單一週期的運作細節。AEE-401 討論的是單步，AEE-701 討論的是整個迴圈。
