# [AEE-200] 模型與上下文層

200 系列文章涵蓋了工程師與模型之間在設定和 API 層面的介面。[基礎與心智模型](../Foundations and Mental Models/100)系列著重建立對 LLM 是什麼、如何推理的直覺；而本層則是將這些直覺落實為工程實踐的地方：Token 化 (tokenization) 如何影響成本與吞吐量、如何有效利用上下文視窗 (context window) 而不造成浪費、如何讓模型輸出可被程式解析的結構化資料，以及如何為手邊的任務挑選合適的模型。

這些主題形成一個緊密的整體，因為它們回答的是同一類問題：給定一個模型和一次 API 呼叫，你手上有哪些調控空間？又該如何正確使用？任一環節的誤解——把 token 數當成概略估算、忽視上下文視窗的結構、讓輸出保持非型別化、或對每個子任務都選用旗艦模型——都會在智能體系統中持續累積影響。把這些做對，是區分「偶爾能跑的原型」與「可靠的生產系統」的關鍵。

## 本類別的文章

**[AEE-201 Token 化實務](201)** — 說明文字如何映射至真實模型中的 token，讓你能預估成本、避免截斷意外，並寫出符合預算限制的提示詞。

**[AEE-202 上下文視窗架構](202)** — 說明上下文如何被組織與消耗，防止你把視窗填滿低價值內容、反而擠掉模型真正需要的資訊。

**[AEE-203 結構化輸出設計](203)** — 示範如何將模型輸出限制為可靠的結構化綱要 (schema)，讓下游程式無須解析自由文字或處理意外格式。

**[AEE-204 系統提示詞工程](204)** — 教你撰寫能在多輪對話中維持穩定行為的系統提示詞 (system prompt)，避免設計不良的智能體常見的漂移與矛盾問題。

**[AEE-205 微調基礎](205)** — 釐清微調 (fine-tuning) 真正有效的時機，以及提示詞工程已足夠應對的情況，讓你只在值得的地方投入訓練運算資源。

**[AEE-206 生產環境的模型選擇](206)** — 提供將模型能力與成本對應至各項任務的決策框架，讓你不再用昂貴模型處理低成本請求。
