# [AEE-0] AEE 總覽

:::info
AEE（Agentic Engineering Essentials，智能體工程精要）是一套與廠商無關的指南集，為建構、運行與治理智能體 AI 系統的工程師提供最佳實踐。
:::

## Context

智能體 AI 工程是一門獨立的學科，有別於機器學習工程與傳統軟體工程。機器學習工程師負責優化模型；軟體工程師構建確定性系統。智能體工程師兩者都不單獨做——他們設計的系統由模型驅動執行、工具擴展能力，成敗取決於模型行為、執行框架架構與任務設計之間的交互作用。

目前可用的學習資源分為兩類：特定於框架的淺層教程，或嚴謹但難以用於生產的學術論文。兩者都無法建立在智能體系統上做出良好架構決策所需的心智模型。

AEE 提供第三條路：以概念而非框架組織的有編號、有結構的文章。每篇文章都經過權威來源的研究驗證，指導原則保持廠商中立，旨在建立對智能體技術棧的完整理解——從智能體是什麼，到如何在生產環境中評估、運維和保護它。

## Purpose

- 建立智能體工程討論的共同詞彙
- 為常見智能體系統決策提供可操作、與廠商無關的指導
- 作為初學者進入智能體 AI 的入門資源
- 彌合「我會提示模型」與「我能建構可靠的智能體系統」之間的差距

## 適合閱讀 AEE 的讀者

AEE 適合以下工程師：

- 正在構建或評估智能體系統，並希望建立結構化心智模型
- 已使用 LLM API 或聊天介面，但尚未構建生產級智能體管線
- 具備傳統軟體或機器學習工程背景，需要理解智能體層
- 希望獲得不受特定框架或模型廠商綁定的通用指導

AEE 不是框架教程，不教導任何特定產品，而是教授支撐所有這些框架的底層概念。

## 如何閱讀 AEE

每篇 AEE 遵循一致的結構：

- **Context（背景）** -- 此主題在智能體工程中的重要性
- **Design Think（設計思考）** -- 核心概念框架；使用 RFC 2119 關鍵字（MUST、SHOULD、MAY）
- **Deep Dive（深入探討）** *（可選）* -- 適合希望深入了解的讀者
- **Best Practices（最佳實踐）** *（可選）* -- 可操作的、與廠商無關的指導
- **Visual（視覺化）** *（可選）* -- 有助理解時提供圖表
- **Examples（範例）** *（可選）* -- 具體的、與廠商無關的範例
- **Related AEEs（相關 AEE）** -- 交叉引用其他原則
- **References（參考資料）** -- 外部資源；所有 URL 均為真實且經過驗證
- **Changelog（更新日誌）** -- 此文章的版本歷史
