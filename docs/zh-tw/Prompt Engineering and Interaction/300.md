# [AEE-300] 提示工程與互動

## 情境

300 系列涵蓋提示工程 (prompt engineering) 與互動——工程師用來向語言模型可靠且可驗證地傳達任務意圖的技術。如果 200 系列（模型與情境層）的重點在於理解模型的運作參數，300 系列則著重於如何有效運用這些參數：如何組織送給模型的內容、如何引導模型進行推理、如何挑選並使用示例、如何撰寫真正被遵循的指令、如何透過聚合提升可靠性，以及如何驗證提示在真實輸入變化下仍能穩定運作。

提示工程有時被視為一種非正式技能——靠試誤累積的直覺。這樣的定位並不完整。能在生產環境中穩定發揮效果的提示，是經過設計的結構化產物：它們有刻意安排的元件佈局、經過輸入變化測試的穩健性，以及與所配置系統並行維護的版本歷史。本系列文章將提示工程視為一門工程學科，具備可重現的技術與可量測的品質標準。

## 涵蓋範圍

300 系列依結構到驗證的順序組織：

**結構與技術文章**涵蓋核心構建塊：
- **[AEE-301](301)** — 提示解剖學：一個完整提示的各個元件，以及它們如何在系統、使用者與助理輪次中相互作用。
- **[AEE-302](302)** — 思維鏈提示 (chain-of-thought prompting)：推理步驟如何提升複雜任務的表現，以及何時無效。
- **[AEE-303](303)** — 少樣本提示 (few-shot prompting)：如何挑選、排序並使用示例，以啟動模型的情境學習能力。
- **[AEE-304](304)** — 指令遵循 (instruction following)：什麼樣的指令能被可靠解析，以及如何找出並修正導致不合規的常見失效模式。

**可靠性文章**涵蓋超越單次生成以提升輸出品質的技術：
- **[AEE-305](305)** — 自一致性與集成 (self-consistency and ensembling)：聚合多次生成結果，以降低高風險決策的變異性。
- **[AEE-306](306)** — 提示穩健性測試 (prompt robustness testing)：在部署前針對輸入變化測試提示，並持續防止回歸。

## 與相鄰範圍的關係

300 系列直接建立在 200 系列（模型與情境層）的基礎上。理解情境視窗的運作方式（AEE-202）、系統提示在架構層面的功能（AEE-204），以及模型如何處理結構化輸出（AEE-203），對本系列的每一項技術都有直接影響。AEE-204（系統提示工程）與 AEE-301（提示結構基礎）關係密切：AEE-204 將系統提示視為一種架構產物，AEE-301 則涵蓋所有輪次的提示解剖學。

400 系列（工具使用與編排）將提示工程應用於更複雜的工具呼叫代理情境。用來配置工具呼叫行為、限制工具選擇，以及格式化工具結果以供下游步驟使用的提示，都建立在本系列所介紹的技術之上。
