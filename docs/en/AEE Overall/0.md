# [AEE-0] AEE Overview

:::info
AEE (Agentic Engineering Essentials) is a collection of vendor-agnostic guidelines and best practices for engineers building, running, and governing agentic AI systems.
:::

## Context

Agentic AI engineering is a rapidly emerging discipline. Engineers building production agent systems must reason across a wide stack: model selection, prompt design, tool integration, memory management, harness architecture, evaluation, operations, safety, and security. Most learning resources are either framework tutorials (vendor-specific, shallow) or research papers (rigorous, impractical). AEE bridges that gap with numbered, structured articles that build a coherent mental model of agentic engineering -- from foundational concepts to production-grade harness design.

## Purpose

- Establish shared vocabulary for agentic engineering discussions
- Provide actionable, vendor-neutral guidance for common agentic system decisions
- Serve as an onboarding resource for engineers new to agentic AI
- Bridge the gap between "I can prompt a model" and "I can build reliable agent systems"

## How to Read AEEs

Each AEE follows a consistent structure:

- **Context** -- Why this topic matters in agentic engineering
- **Design Think** -- Core conceptual framing; uses RFC 2119 keywords (MUST, SHOULD, MAY)
- **Deep Dive** *(optional)* -- Technical depth for readers who want to go further
- **Best Practices** *(optional)* -- Actionable, vendor-neutral guidance
- **Visual** *(optional)* -- Diagrams where they aid understanding
- **Examples** *(optional)* -- Concrete, vendor-agnostic examples
- **Related AEEs** -- Cross-references to other principles
- **References** -- External resources; all URLs are real and verified
- **Changelog** -- Version history for this article
