# [AEE-0] AEE Overview

:::info
AEE (Agentic Engineering Essentials) is a collection of vendor-agnostic guidelines and best practices for engineers building, running, and governing agentic AI systems.
:::

## Context

Agentic AI engineering is a distinct discipline from both machine learning engineering and traditional software engineering. ML engineers optimize models; software engineers build deterministic systems. Agentic engineers do neither exclusively -- they design systems where a model drives execution, tools extend capability, and success depends on the interaction between model behavior, harness architecture, and task design.

The learning resources available today fall into two categories: framework tutorials that are vendor-specific and shallow, or research papers that are rigorous but impractical for production use. Neither builds the mental model needed to make sound architectural decisions about agent systems.

AEE provides a third path: numbered, structured articles organized by concept, not by framework. Each article is researched against authoritative sources, vendor-neutral in guidance, and designed to build a coherent understanding of the agentic stack -- from what an agent is to how to evaluate, operate, and secure one in production.

## Purpose

- Establish shared vocabulary for agentic engineering discussions
- Provide actionable, vendor-neutral guidance for common agentic system decisions
- Serve as an onboarding resource for engineers new to agentic AI
- Bridge the gap between "I can prompt a model" and "I can build reliable agent systems"

## Who Should Read AEE

AEE is written for engineers who:

- Are building or evaluating agentic systems and want a structured mental model
- Have used LLM APIs or chat interfaces but have not yet built production agent pipelines
- Are experienced in traditional software or ML engineering and need to reason about the agentic layer
- Want vendor-neutral guidance that transfers across frameworks and model providers

AEE is not a framework tutorial. It does not teach any specific product. It teaches the concepts that underlie all of them.

## How to Read AEEs

Each AEE follows a consistent structure:

- **Context** -- Why this topic matters in agentic engineering
- **Design Think** -- Core conceptual framing; uses RFC 2119 keywords (MUST, SHOULD, MAY)
- **Deep Dive** *(optional)* -- Technical depth for readers who want to go further
- **Best Practices** *(optional)* -- Actionable, vendor-neutral guidance
- **Visual** *(optional)* -- Diagrams where they aid understanding
- **Examples** *(optional)* -- Concrete, vendor-agnostic examples
- **Related AEEs** -- Cross-references to other principles
- **References** -- External resources; all URLs are real and verified
- **Changelog** -- Version history for this article
