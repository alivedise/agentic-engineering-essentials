# [AEE-300] Prompt Engineering & Interaction

## Context

The 300-range covers prompt engineering and interaction — the techniques engineers use to communicate task intent to a language model reliably and verifiably. If the 200-range (Model & Context Layer) is about understanding the model's operational parameters, the 300-range is about using them effectively: how to structure what you send to the model, how to elicit reasoning, how to select and use examples, how to write instructions that are actually followed, how to improve reliability through aggregation, and how to verify that prompts hold up under real-world input variation.

Prompt engineering is sometimes framed as an informal skill — intuition refined through trial and error. That framing is incomplete. Prompts that work reliably in production are structured artifacts: they have deliberate component layout, tested robustness across input variation, and version history alongside the systems they configure. The articles in this range treat prompt engineering as an engineering discipline with reproducible techniques and measurable quality criteria.

## What This Range Covers

The 300-range is organized as a stack from structure to validation:

**Structure and technique articles** cover the building blocks:
- **[AEE-301](301)** — prompt anatomy: the components of a well-formed prompt and how they interact across system, user, and assistant turns.
- **[AEE-302](302)** — chain-of-thought prompting: how reasoning steps improve performance on complex tasks, and when they do not.
- **[AEE-303](303)** — few-shot prompting: how to select, order, and use examples to activate the model's in-context learning.
- **[AEE-304](304)** — instruction following: what makes instructions parse reliably, and how to find and fix the failure patterns that cause non-compliance.

**Reliability articles** cover techniques for improving output quality beyond a single completion:
- **[AEE-305](305)** — self-consistency and ensembling: aggregating multiple completions to reduce variance on high-stakes decisions.
- **[AEE-306](306)** — prompt robustness testing: testing prompts against input variation before deployment and maintaining them against regression.

## How This Range Relates to Adjacent Ranges

The 300-range builds directly on the 200-range (Model & Context Layer). Understanding how context windows work (AEE-202), how system prompts function architecturally (AEE-204), and how models process structured output (AEE-203) informs every technique in this range. AEE-204 (System Prompt Engineering) and AEE-301 (Prompt Structure Fundamentals) are closely related: AEE-204 covers the system prompt as an architectural artifact, while AEE-301 covers prompt anatomy across all turns.

The 400-range (Tool Use & Orchestration) applies prompt engineering to the more complex context of tool-calling agents. Prompts that configure tool-calling behavior, constrain tool selection, and format tool results for downstream steps all draw on the techniques in this range.
