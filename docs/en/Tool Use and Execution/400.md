# [AEE-400] Tool Use & Execution

## Context

The 400-range is where models stop generating text and start taking actions. It covers the mechanics of how agents invoke external capabilities — the function calling protocol, how to design schemas models follow reliably, sandboxed code execution, browser and computer use automation, how models select among multiple tools, and how to isolate agent execution to control blast radius.

Every technique in the 300-range (Prompt Engineering) produces text. The 400-range produces effects: API calls, database writes, code execution, form submissions, file operations. That distinction matters for safety, for cost, and for system design. Text generation is reversible; tool execution often is not. Understanding how tools work, how models select them, and how to bound their impact is not optional for production agent systems.

## What This Range Covers

The 400-range is organized from protocol to safety:

**Protocol and schema articles** cover the function calling mechanism:
- **[AEE-401](401)** — function calling: the full API protocol from tool definition to result injection, with concrete Anthropic API examples.
- **[AEE-402](402)** — tool schema design: how to write names, descriptions, and parameter schemas that models parse and invoke correctly.

**Tool type articles** cover the two highest-capability tool categories:
- **[AEE-403](403)** — code execution: sandboxed interpreters, provider options, and the security requirement that model-generated code never executes in the host process.
- **[AEE-404](404)** — browser and computer use: web automation with Playwright and browser-use, and Anthropic's computer use API for agents that interact with full desktop environments.

**Selection and safety articles** cover the cross-cutting concerns:
- **[AEE-405](405)** — tool selection: why models choose the wrong tool, how description quality drives selection accuracy, and how to manage large tool sets.
- **[AEE-406](406)** — sandboxing and execution safety: blast radius, the principle of least privilege applied to tools, isolation layers, and human-in-the-loop gates.

## How This Range Relates to Adjacent Ranges

The 400-range builds on the 300-range (Prompt Engineering & Interaction). Tool descriptions are prompts: the principles of instruction writing (AEE-304), schema design (AEE-203), and robustness testing (AEE-306) all apply to how tools are described to models. The 300-range techniques determine whether models use tools correctly; the 400-range mechanics determine whether tool execution is safe.

The 400-range connects upward to the 700-range (Agent Loops and Orchestration). AEE-701 (The Agent Loop / ReAct) covers how tool calls fit into a multi-step reasoning loop — observation, reasoning, action sequencing. AEE-401 (Function Calling) covers the mechanics of a single cycle within that loop. AEE-401 is about one step; AEE-701 is about the loop.
