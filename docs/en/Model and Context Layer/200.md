# [AEE-200] Model & Context Layer

The 200-range articles cover the interface between you and the model at the configuration and API level. Where the [Foundations and Mental Models](../Foundations and Mental Models/100) series builds intuition about what LLMs are and how they reason, this layer is where that intuition becomes engineering practice: how tokens map to cost and throughput, how to pack context windows without waste, how to coerce models into producing machine-readable output, and how to choose the right model for the workload in front of you.

These topics form a coherent unit because they all answer the same class of question: given a model and an API call, what levers do you have, and how do you use them correctly? Misunderstanding any one of them — treating token counts as approximate, ignoring context window structure, leaving outputs untyped, or picking a flagship model for every subtask — compounds across an agentic system. Getting them right is the difference between a prototype that occasionally works and a production system that reliably does.

## Articles in This Category

**[AEE-201 Tokenization in Practice](201)** — Explains how text maps to tokens in real models so you can predict costs, avoid truncation surprises, and write prompts that fit within budget.

**[AEE-202 Context Window Architecture](202)** — Covers how context is structured and consumed, preventing the common mistake of filling the window with low-signal content and crowding out what the model actually needs.

**[AEE-203 Structured Output Design](203)** — Shows how to constrain model output to a reliable schema so downstream code never has to parse free text or handle unexpected formats.

**[AEE-204 System Prompt Engineering](204)** — Teaches how to write system prompts that establish stable behavior across turns, avoiding the drift and contradiction that plague poorly specified agents.

**[AEE-205 Fine-Tuning Basics](205)** — Clarifies when fine-tuning actually helps versus when prompt engineering is sufficient, so you invest training compute only where it earns a return.

**[AEE-206 Model Selection in Production](206)** — Provides a decision framework for matching model capability and cost to each task, so you stop over-serving cheap requests with expensive models.
