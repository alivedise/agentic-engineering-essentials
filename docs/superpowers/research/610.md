## Research notes

- **Status: active, not absorbed.** Unlike ACP (which was folded into A2A in August 2025), AG-UI remains an independent project with its own org (`ag-ui-protocol`), its own docs site (docs.ag-ui.com), and continued first-party adapter releases through 2026. No merger or deprecation flag found.
- **Naming.** "AG-UI" is the canonical name. "A2UI" is a separate, unrelated specification originated by Google for declarative generative UI. CopilotKit explicitly published a comparison page distinguishing the two; the article should call out that they are complementary and not synonyms.
- **Event count caveat.** CopilotKit's blog headlines "17 AG-UI event types," but the canonical `EventType` enum in the JS SDK currently lists 26 values once the reasoning extension is included (RUN/STEP × 5, TEXT × 3, TOOL × 4, STATE × 3, ACTIVITY × 2, RAW + CUSTOM, plus 7 REASONING events). Safer phrasing in the article: "AG-UI defines a small set of standardized event types — lifecycle, text-message, tool-call, state, and reasoning families — currently around two dozen events in the reference enum." Avoid pinning a specific count without dating it.
- **Run vocabulary.** AG-UI uses `Run` (with `threadId` + `runId`) as the unit of agent execution, conceptually parallel to A2A's `Task` and ACP's `Run`. Worth explicitly drawing this parallel in the Related AEEs cross-link.
- **Governance.** No formal foundation governs AG-UI as of the sources fetched; it is "developed by the CopilotKit team in collaboration with leading frameworks." Treat as vendor-led-but-open, comparable to MCP's pre-foundation phase. Don't overstate neutrality.
- **Letta integration not confirmed.** The brief listed Letta as an expected integration; the official partner table at docs.ag-ui.com does not include Letta. Drop Letta from the article's adopter list and use the verified set above.
- **SDK naming.** TypeScript SDK ships as `@ag-ui/core` plus `@ag-ui/client` and per-framework packages like `@ag-ui/mastra`, `@ag-ui/langgraph`. Python SDK is the `ag_ui` namespace (`ag_ui.core`, `ag_ui.encoder`); a separate `ag-ui-protocol` PyPI name was not surfaced in the docs.
