- **Target section:** Deep Dive

## Research notes

- **Domain migration:** During verification, every `a2aproject.github.io/A2A/...` URL 301-redirected to `a2a-protocol.org/...`. The article's References section should use the post-redirect canonical URLs (`a2a-protocol.org`) listed above to avoid the redirect hop. The `github.com/a2aproject/A2A` repo URL is unchanged.
- **Spec version churn:** The official spec page now lists v1.0.0 as latest (released March 12, 2026 per the GitHub release history), with v0.3.0 still referenced as the most recent "stable enterprise" milestone in the Google Cloud blog. The Python SDK supports 1.0 (with 0.3 compatibility); the JS SDK README still references 0.3.0. Worth noting in the article that v1.0 is very recent and SDKs are still catching up.
- **Lifecycle states caveat:** The `life-of-a-task` page enumerates terminal/interrupted states explicitly (`completed`, `canceled`, `rejected`, `failed`, `input-required`, `auth-required`) but does not enumerate intermediate states like `submitted` or `working` in the page body — those appear in the spec proto file. The brief in this task mentioned "submitted → working → input-required → completed/failed/canceled," but `rejected` and `auth-required` are also part of the current spec. The writer should use the verified state list from this findings doc rather than the brief's outdated list.
- **Part naming caveat:** The brief used names like "TextPart, FilePart, DataPart" (matching some SDK type names and earlier docs/blog posts), but the v1.0 spec uses generic field names `text`, `raw`, `url`, and `data` within a unified `Part` structure. The article can use either form but should flag that the SDK class names (TextPart/FilePart/DataPart) are SDK-side conveniences, while the wire format uses field discriminators on a single `Part` type.
- **Transport bindings:** The brief described A2A as JSON-RPC 2.0 only, but v1.0 introduces three bindings (JSON-RPC, gRPC, REST). The article's Wire Format section should lead with JSON-RPC 2.0 (most common) but mention gRPC and REST exist.
- **Adoption claim:** Verified primary-source counts are "more than 50 partners" at launch (April 2025) and "more than 100 companies" by the Linux Foundation announcement (June 2025) and "over 150 organizations" by August 2025. Stick to the LF founding members (AWS, Cisco, Google, Microsoft, Salesforce, SAP, ServiceNow) for the named-adopters claim — primary-sourced and stable.
- **MCP comparison source:** The official `a2a-and-mcp` page handles this contrast cleanly with the auto-repair-shop analogy. AEE-609 (ACP) comparison is not covered by primary A2A docs and will need its own research in the AEE-609 findings.
- **AG-UI comparison:** Out of scope for A2A primary sources; cross-reference to AEE-610 only, no claim sourced here.
